Microsoft: KB5021233 вызывает синий экран с ошибками 0xc000021a
Серджиу Гатлан

    19 декабря 2022 г., 07:19 9

Windows 10

Microsoft исследует известную проблему, приводящую к cинему экрану смерти (BSOD) с ошибками 0xc000021a после установки накопительного обновления Windows 10 KB5021233, выпущенного во время вторничных обновлений в этом месяце.

На выходных компания предупредила, что «после установки KB5021233 некоторые устройства Windows могут запуститься с ошибкой 0xc000021a на синем экране».

Эта известная проблема, вероятно, вызвана несоответствием между версиями файла hidparse.sys в system32 и system32/drivers в папке Windows, «что может привести к сбою проверки подписи при очистке».

Список уязвимых платформ включает только клиентские версии Windows 10, начиная с Windows 10 20H2 и заканчивая последней версией Windows 10 22H2.

Редмонд добавил, что уже работает над исправлением этой недавно выявленной проблемы, но пока не сообщил, когда оно будет доступно.

Решение для пораженных систем Windows 10

Тем не менее, компания поделилась временным решением проблемы с синим экраном и оживлением пораженных систем, чтобы они запускались нормально.

Для этого пользователям необходимо скопировать файл hidparse.sys из папки system32\drivers в system32 через Среду Восстановления Windows (WinRE), чтобы проверка подписи больше не завершалась ошибкой во время очистки.

Чтобы устранить эту проблему на пораженных устройствах, вам необходимо выполнить следующие шаги:

    Вам нужно будет войти в Среду Восстановления Windows. Если ваше устройство не запустилось в WinRE автоматически, см. Точки входа в WinRE.
    Нажмите кнопку «Устранение неполадок».
    Нажмите кнопку «Начать восстановление, устранение неполадок и средства диагностики».
    Выберите кнопку «Дополнительные параметры».
    Нажмите кнопку «Командная строка» и дождитесь перезагрузки устройства, если это необходимо.
    Ваше устройство должно перезагрузиться в окне командной строки. Возможно, вам потребуется войти в систему с помощью пароля, прежде чем вы попадете в окно командной строки.
    Выполните следующую команду (Важно: если Windows не установлена в C:\windows, вам нужно будет изменить команду для вашей среды): xcopy C:\windows\system32\drivers\hidparse.sys C:\windows\system32\hidparse .sys
    После завершения предыдущей команды введите: exit
    Нажмите кнопку «Продолжить».
    Теперь Windows должна запуститься, как и ожидалось.

«Не рекомендуется использовать какие-либо другие решения, кроме рекомендованных выше. Мы не рекомендуем удалять файл hidparse.sys из папки Windows\System32», — добавили в Microsoft.

Редмонд также исследует известную проблему, связанную с обновлениями Windows Server Patch, выпущенными в этом месяце во вторник, которая вызывает ошибки при создании новых виртуальных машин на некоторых хостах Hyper-V.
